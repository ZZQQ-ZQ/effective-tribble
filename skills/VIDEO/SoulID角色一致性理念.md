# Soul ID 角色一致性理念（Seedance适配版）

> 源概念：Higgsfield AI `Soul ID` — 上传20+张照片训练角色数字分身，跨场景生成时保持角色一致
> 提取日期：2026-07-13
> 适配前提：Seedance/即梦不支持角色训练，但有@参考图锚定机制

## 核心可迁移理念

### 1. Identity vs. Motion 分离（已存在，强化表述）

**角色身份（Identity）** 和 **动作行为（Motion）** 是两个独立维度，永远不在同一句提示词中混淆：

```diff
- ❌ "秦风穿着黑色战袍，眼神坚毅，右手挥剑击退敌人"
+ ✅ 身份由@参考图锁定 → 提示词只写：秦画右，右手持剑，蹬地冲出，剑刃横扫
```

导演已确立的规则「有参考图只写位置+朝向+动作状态」就是此理念的精髓实现。

### 2. Character Anchor Block（提示词中的角色锚块）

每段提示词的人物主体部分，用固定句式写角色，形成"锚块"——变化的部分永远是后面的动作，前面不变：

```markdown
主体：
秦（右手持剑）        ← 锚块：武器固定
晚晴（右手持刀）      ← 锚块：武器固定
```

锚块在各段之间完全不改，全部变化放在分镜内容中。这相当于Seedance版的"角色身份模块"。

### 3. 跨段落一致性检查

从Soul ID理念延伸出的检查维度：

| 维度 | 检查点 |
|------|--------|
| 武器持手 | 秦→右手剑，晚晴→右手刀，整部剧不变 |
| 身高比例 | 秦和晚晴的身高差在远景镜中保持一致 |
| 武器形态 | 剑/刀的外形不因运镜变形，材质反光一致 |
| 站立姿态 | 无动作时的默认站姿不漂移 |

### 4. 与现有技能的分工

| 技能 | 管什么 |
|------|--------|
| 场景一致性保持技能 | 空间、光线、物体、环境 |
| 人物表演管理技能 | 表情、语气 |
| **Soul ID（本理念）** | 角色身份/装备/武器的跨段落不变性 |
| @参考图 | 外貌、服装、材质（由Seedance平台直接锚定） |

## 实操规则

- 每集剧本写完后，提取所有角色的"固定信息清单"（武器/持手/默认站姿/身高关系）
- 每段提示词的主体行完全复用固定信息，一字不改
- 段落间检查：武器持手未换、默认姿态一致、@参考图引用正确
