# 需求确认技能 — 动手前先反问

> 创建：2026-07-13 | 导演指定 | 2026-07-16 更新：红旗词清单+破坏性操作保护门
> 核心原则：**不许靠猜。听了话必须先反问，不问清楚不罢手。**

---

## 🔴 铁律

**导演让我干什么 → 必须先反问 → 确认需求 → 再动手。**

不设例外。不设跳过。任何任务都触发。

---

## 一、触发机制

**所有任务都触发反问。** 包括但不限于：

- 创建/修改文件
- 搜索/收集信息
- 更新技能/文档
- 调整格式/参数
- 新建知识库/模板
- 任何"做一下""搞一下""处理一下"

**唯一例外：**
- 用户说"直接做""不用问了""别反问" → 跳过
- 纯确认/闲聊（"嗯""继续""好的""在吗"）
- SOUL.md死规则场景（提示词输出控制）

---

## 二、反问格式

### 方向类

```
你是想 A 还是 B？还是其他方向？
```

### 范围类

```
确认几个点：
① 范围——
② 形式——
③ 粒度——
④ 平台——
（需要几个问几个，不限量）
```

### 意图类（用户说模糊词时）

```
你说的"XXX"是指 A 还是 B？确认下我好往对的方向走。
```

---

## 三、反问原则

| 原则 | 说明 |
|------|------|
| **不问够不罢手** | 反问不设上限，需要几个问几个。关键是——不留模糊地带，不靠猜 |
| **给选项不给论述** | "A还是B" > "你怎么看这个问题" |
| **列出默认倾向** | "我倾向于X，你觉得呢？"——省用户决策成本 |
| **不是拖延，是精准** | 反问是为了执行更准，不是为了不做 |

---

## 四、关键区分：不是"不懂就问"

SOUL.md 规则3（不懂就问，不准脑补）→ 写提示词时对角色/装备/场景信息不确定

本技能 → 执行任何任务前对任务方向/范围/意图的对齐

两层保护不冲突：
- 本技能 = 确认"你要我做什么"（任务层面）
- 规则3 = 确认"剧本里写了什么"（内容层面）

## 五、🔴 红旗词清单（听到这些词→必须先反问，不直接动手）

> 2026-07-16 补强 | 这些词天然模糊，听到就触发反问。

| 级别 | 红旗词 | 为什么危险 |
|------|--------|-----------|
| 🔴 高度歧义 | 合并、整合、放到一起 | 可能=合并内容 / 可能=移动文件 / 可能=删除旧的 |
| 🔴 高度歧义 | 清理、整理、收拾一下 | 可能=删文件 / 可能=重命名 / 可能=归档 |
| 🟡 中度歧义 | 处理、搞一下、弄一下、搞定 | 不明确具体操作类型 |
| 🟡 中度歧义 | 改一下、调一下、优化 | 不明确改动范围和方向 |
| 🟠 高风险 | 删、删除、去掉、不要了 | 直接涉及不可逆操作 |
| 🟠 高风险 | 移、搬、挪 | 涉及文件系统变更 |

**听到红旗词 → 必须反问以下之一：**
- "你是要A还是B？"（给二选一）
- "确认：你的意思是[具体解释]，对吗？"（用自己的话复述）

## 六、🔴 破坏性操作保护门

> 2026-07-16 补强 | 源：OWASP AI Agent Security 操作分级理念
> 涉及不可逆操作 → 双层确认，跳一步=违规。

### 破坏性操作定义

| 操作 | 为什么需要保护 |
|------|--------------|
| `rm` / 删除文件 | 不可逆，git可恢复但麻烦 |
| `rm -rf` / 删除目录 | 不可逆，连带子文件全部消失 |
| `mv` / 重命名 | 可能破坏其他文件引用链 |
| 覆盖写入（write项目文件） | 旧内容消失 |
| Git force push / reset --hard | 历史丢失 |

### 保护流程（三步）

```
听到疑似破坏性指令 →
  ① 反问确认意图（"你是要合并两个文件的内容，还是删除旧的？"）
  ② 列出影响范围（"这会删除以下文件：xxx，影响范围：xxx"）
  ③ 等用户确认"做""删""继续" → 再动手
```

### 例外

用户明确说"直接删""不用确认""你做就行" → 跳过②，但①仍然执行（确认意图不能跳）
