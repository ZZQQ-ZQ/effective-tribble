# 场景一致性保持技能

> **创建日期**：2026-06-28
> **来源**：四面墙方法（本地）+ Seedance 2.0官方/社区最佳实践（Scenario/InVideo/educba/Vmake/Reddit）+ 八维一致性准则（USER.md）+ 多镜头一致性工作流（prompt-optimizer 30.69.13-14）
> **用途**：确保 Seedance 视频提示词的场景在跨镜头、跨片段、跨集生成中保持空间/视觉/光影/物体一致性，防止场景漂移
> **与现有体系关系**：本技能独立覆盖「场景一致性」，与「人物服装增强技能」（管人物质感）、「场景质感增强技能」（管场景材质纹理）互补。写提示词时三技能同时参考。

## 一、场景漂移诊断

### 常见漂移表现

| 漂移类型 | 表现 | 根因 |
|---------|------|------|
| 空间漂移 | 家具位置变、墙颜色变、房间结构变 | 无空间锚点，模型每次重新「想象」空间 |
| 光影漂移 | 光源方向跳变、色温突变、阴影位置不连续 | 未在每镜锁定光源方向和色温 |
| 物体漂移 | 道具消失/变形/颜色变、装饰物换位 | 物体未以参考图锚定 |
| 环境漂移 | 室外转室内、天气变、地面材质变 | 场景段未统一环境标签 |
| 比例漂移 | 人物与空间大小关系不协调 | 无统一尺度参照，模型自由发挥 |

### 核心原则

**场景一致性 = 空间锚定 × 光影锁定 × 物体参考 × 环境统一描述。** 四个维度缺一不可。

### 光线漂移三大触发点（必须避开）

> 来源：Forasoft 2026 生产一致性分析 + PixVerse 7项实测

**触发1：镜头运动导致模型重绘背景**
镜头每动一次，模型就要"猜测"画面外的新内容 → 光源猜测不一致 → 光线漂移。
> 对策：同场景内保持镜头运动一致，不要这镜固定、下镜环绕。避免堆叠多种镜头运动。

**触发2：色温变化导致人脸重绘**
从暖光切到冷光时，模型把"暖光下的脸"和"冷光下的脸"当成两个不同东西来画 → 脸变。
> 对策：同场景绝不改色温。要改就在场次切换时改。"同一张脸在不同光下是同一个东西"对模型不是默认成立的。

**触发3：模糊形容词让模型自由发挥**
"戏剧性灯光"可以是顶光强对比、也可以是侧光长阴影、也可以是逆光剪影——每次生成随机选一种。
> 对策：永远用具体物理光源描述，永远不用形容词。用"2700K钨丝灯"替代"温暖的灯光"。

## 二、四面墙空间锚定法（空间逻辑）

> 来源：四面墙方法原始文档（已整合去重）

### 原理

把场景想象成一个房间的四堵墙。所有镜头机位、人物移动、视线方向必须遵守这个空间布局。不写逻辑自相矛盾的描述。

### 步骤1：确定场景空间结构

写分镜前，先在脑子里画场景平面图，标记四向特征和关键物体位置：

```
【户外场景版——以古建筑为例】
- 北（远）：主殿/山门/远景标志物
- 南（近·镜头方向）：观众所在位置
- 东（左/画右）：侧殿/廊道/树木
- 西（右/画左）：围墙/演武台/石阶
```

```
【室内场景版——以客厅为例】
- 北墙：窗户/光源
- 南墙：电视/装饰画
- 东墙：沙发/茶几
- 西墙：门/柜子
```

### 步骤2：标记关键锚点

每个场景只需锁定 3-5 个视觉锚点，不需要每个物体都写。锚点的作用是让模型「认得出这是同一个地方」。

**锚点选择原则**：选最大、最独特、最容易在画面中出现的元素。
- ✅ 好锚点：主殿红柱金瓦、歪脖子树、石质演武台、巨大落地窗
- ❌ 差锚点：角落小花瓶、地板纹理方向、天花板灯饰

**每镜内容中至少出现 1-2 个锚点元素**，观众潜意识里就认得出是同一个场景。

### 步骤3：建立空间逻辑规则

每种场景类型有不同的核心逻辑约束：

| 场景类型 | 核心约束 |
|---------|---------|
| 室内对话 | 视线方向一致、家具位置不变、光源方向统一 |
| 户外打斗 | 地面材质不变、远景标志物方位一致、天气光线统一 |
| 多人对峙 | 站位关系锁定、遮蔽关系合理、移动路径不穿模 |
| 追逐跑动 | 路径连贯、地标依次出现、速度匹配距离 |

### 步骤4：空间逻辑快速检查

写完全部分镜后，逐镜回看：

- [ ] 光源方向是否每镜一致？（太阳/主灯从哪边来）
- [ ] 地标/锚点元素是否至少出现了1次？
- [ ] 人物移动路径是否连贯不跳跃？
- [ ] 远景和近景的空间关系是否匹配？（远景看到山→近景窗外应该是山）
- [ ] 遮挡关系是否合理？（沙发后的人突然全身可见→穿帮）

### 步骤5：人物朝向与机位空间分离（防正面冲镜）

> 来源：2026-07-04 导演实战反馈——多个段反复出现人物正面面对镜头的画面，根因是提示词中"面朝XXX"与相机机位未做空间分离

#### 核心问题

AI 视频模型（包括 Seedance 2.0）没有三维空间推理能力。当提示词写「秦风面朝BOSS」，而相机视角方向恰好与人物→BOSS连线方向一致时，模型看到的画面就是：一个人正面对镜头。它无法判断「这个人是看BOSS不是看相机」。

#### 原理：人物朝向和机位是两条独立轴线

```
【错误——模型视角】
人物 → 面朝BOSS → 相机在该方向 → 人物面朝镜头 ❌

【正确——两条轴线分开写】
轴线1：人物身体朝向 → 指向BOSS方向（画右）
轴线2：相机机位 → 在人物侧面/身后/头顶
两条线不重合 → 模型能理解人物不是在看镜头 ✅
```

#### 三法则

**法则一：人物朝向+机位方位必须绑定在同一句写**

每镜写人物动作时必须同时声明两件事：
1. 人物身体面向哪个三维方向
2. 相机相对于人物脸的方位

| 写法 | 模型解读 |
|------|---------|
| ❌ `秦风面朝BOSS` | 脸朝前（前=镜头默认方向）→ 冲镜 |
| ❌ `秦风面朝BOSS，侧面平拍` | 两句分开，模型可能不关联 |
| ✅ `秦风身体侧对镜头面朝前方BOSS方向，机位在秦风右侧` | 朝向+机位在一句，模型连接理解 |

**法则二：多用3/4角度和纯侧面，少用无修饰的正面**

| 术语 | 模型识别度 | 效果 |
|------|-----------|------|
| `侧面轮廓` / `纯侧面` | ⭐⭐⭐⭐⭐ | 脸完全不对镜头 |
| `四分之三侧面` / `3/4 angle` | ⭐⭐⭐⭐ | 脸偏转，不对镜头 |
| `背影朝镜头` / `back to camera` | ⭐⭐⭐⭐ | 完全背对 |
| `过肩` / `over shoulder` | ⭐⭐⭐⭐⭐ | 前景遮挡，脸不对镜头 |
| `后脑勺朝镜头` / `facing away` | ⭐⭐⭐⭐ | 明确背向 |
| ❌ 只写 `正面平拍` 无朝向修饰 | ⭐ | 大概率冲镜 |

**法则三：镜头是观察者，不是对话参与者**

写内容时把镜头当成「旁观的眼睛」，不让人物有「展示/面向给镜头看」的感觉。

| 感觉 | 写法 |
|------|------|
| ❌ 表演感 | `秦风正面看向镜头方向，嘴角扬起` |
| ✅ 旁观感 | `秦风侧脸轮廓在暖金日光中收紧，目光锁定画右前方BOSS` |
| ❌ 展示感 | `晚晴正对镜头挥刀劈下` |
| ✅ 旁观感 | `机位在晚晴右侧——她身体前倾侧面挥刀，刀锋余光从画左切入画右` |

#### 每镜出稿前自问

- [ ] 这镜人物脸朝向哪里？（BOSS/同伴/地面/天空）
- [ ] 相机放在哪里？（左/右/后/上/下/过肩）
- [ ] 脸朝向和相机方向重合了吗？（重合=冲镜，必须分叉）

#### 典型错误模式速查

| 常见写错 | 为什么会导致冲镜 | 怎么改 |
|---------|----------------|--------|
| 「秦风面朝BOSS，MS，固定」 | 没说相机在哪，默认识别为正前方 | 加`机位在秦风右侧，侧面平拍` |
| 「晚晴正面平拍」 | 直接指定正面 | 改`机位在晚晴右侧，3/4侧面` |
| 「两人正面朝向BOSS，镜头前推」 | 脸朝前+镜头前推=冲镜 | 改`两人面向画右BOSS方向，机位在两人左侧跟拍` |
| 「pov切秦风第一视角看BOSS」 | POV例外，这是"通过人物眼睛看" ✅ | 保持不变 |

## 🔴 二A、全局主体块 vs 每镜独立空间定位（2026-07-06 补入）

> 来源：全网搜索（Seedance 2.0 官方/社区/实践） + 导演实战踩坑

### 核心问题

**在提示词「主体」段全局锁定角色站位（画左/画右/画中）会导致跨镜空间逻辑崩溃。**

根因：AI 视频模型（包括 Seedance 2.0）**没有跨镜空间记忆**——每段提示词被当作独立生成任务执行。模型不会「记住」上一镜的空间关系，因为每次生成都是从零开始的扩散过程。

### 全网证据链

#### 证据1：Seedance 2.0 是「分镜执行器」不是「故事理解器」

Medium 核心教程明确指出 Seedance 2.0 的 Timeline Prompting 结构：

```
[0:00-0:05] Subject: empty diner, fluorescent light flickering. Camera: static wide shot.
[0:05-0:10] Subject: hands wrapping around a coffee mug, steam rising. Camera: close-up, shallow focus.
[0:10-0:15] Subject: second figure entering through the door, backlit. Camera: slow dolly out.
```

**每个时间戳有自己独立的 Subject 描述，位置、动作、相机全部逐镜重写。** 官方定义：「This is closer to a shot list than a description. The model isn't improvising a story, it's executing a plan.」

#### 证据2：AI 模型没有方位意识

即梦社区实战总结（币安/知乎多篇教程一致）：
> 「AI 没有方位意识，经常分不清左右，需要另外做位置关系参考图告诉AI人物是怎么运动的。」

每镜提示词结构被确认为：**画面主体 + 所在位置 + 正在做什么 + 拍摄角度** — 四个要素全部逐镜写，不做全局预声明。

#### 证据3：Cliprise 生产指南的每镜独立规则

Seedance 2.0 生产级 Prompt 公式：`[Subject + 具体细节] + [一个具体动作] + [环境] + [一个相机运动+构图] + [光源+氛围] + [风格锚]` — 没有全局站位字段。每镜的 Subject 段自己描述该镜中主体的位置、状态、朝向。

#### 证据4：阿里云文生视频官方结构

阿里云官方多镜头提示词规范：`总体描述 + 镜头序号 + 时间戳 + 分镜内容`。**每个镜头独立写分镜内容**，不做全局位置声明。

### 为什么全局锁站位会出问题

```
【错误写法】
主体：秦风（画左），晚晴（画右）

镜1：全景，两人对峙
镜2：秦风转身走向通道深处  ← 秦风移动了，但「画左」被全局锁定
镜3：晚晴正面镜头        ← 只有晚晴一人，画左/画右失去参照
镜4：背后跟拍两人        ← 机位变到身后，画左画右全反了

→ 结果：每一镜都有「画左」「画右」约束，但模型无法理解「这张镜头的画左是从哪个方向拍」
→ 空间逻辑彻底崩塌
```

**全局站位只在一种情况下安全：所有镜头同一机位、同一角度、人物不动。** 这在任何正常叙事中都不成立。

### 正确做法

#### 规则1：主体段只声明「谁在场 + 持什么武器」，不写位置

```
【正确写法】
主体：
秦风（右手垂握刀）
晚晴（右手垂握剑）
```

主体段是「演员表」而非「站位表」——告诉模型这场戏有谁，空间关系留给每镜自己去构建。

#### 规则2：每镜「构图」+「画面内动」自己写位置+朝向+机位绑定

```
镜1（0-4秒）：
构图：秦风前景画左（背对镜头），晚晴中景画右（面对镜头），通道向后延伸
画面内动：秦风右手垂握刀，侧头回看晚晴。

镜2（4-8秒）：
构图：秦风背影占画中偏右，正朝通道深处走去；晚晴前景画左下方，肩部虚焦
画面内动：秦风迈步走向通道深处未受光区域，身影逐渐缩小。晚晴微转头斜看秦风方向。
```

每镜的位置描述绑定到**该镜的特定相机机位**，不跨镜复用。

#### 规则3：位置用「构图语言」而非「绝对坐标」

| ❌ 绝对坐标 | ✅ 构图语言 |
|------------|-----------|
| 画左 / 画右（无上下文） | 前景画右 / 中景偏左 / 背景中央 |
| 面朝左 / 面朝右 | 面朝通道深处 / 看向同伴 / 身体侧对镜头 |
| 在左边 / 在右边 | 机位在X左侧，3/4侧面 / 过X右肩，X右肩在左下角虚焦 |

#### 规则4：机位和人物朝向必须一句绑定写

和 二·步骤5 的「人物朝向与机位空间分离」规则一致：
- 每镜的「构图」写清楚机位方位和人物在画面中的空间位置
- 每镜的「画面内动」写清楚人物面朝方向
- 两者在一镜内绑定，不跨镜依赖

### 出稿前自检

写完全部分镜后，逐镜检查：
- [ ] 主体段是否只写了角色名+武器，没有写画左/画右/画中？
- [ ] 每镜的「构图」字段是否独立写了该镜的空间位置？
- [ ] 每镜的「画面内动」字段是否写了朝向和状态？
- [ ] 是否有任何镜头的空间描述依赖于其他镜头的上下文？
- [ ] 如果删掉其他所有镜头只留这一镜，这镜的空间描述是否依然完整自洽？

## 三、八维跨镜一致性准则

> 来源：USER.md 八维一致性准则（2026-04-27）+ 落地经验完善

写分镜时必须逐维检查：

### 维度1：人物一致
角色必须严格匹配角色信息库和参考图。有参考图的角色不写外貌描述，只写位置和动作状态。

### 维度2：时空一致 + 🔴 光线锁死
相邻分镜的时间、地点、光线必须无缝衔接。
- 同一片段内时间不跳跃（不能前一镜白天下一镜黄昏除非剧情需要）
- 光源方向和色温必须统一（判断标准：所有镜头主光源来自同一方向）

**🔴 光线锁死四要素（每条提示词必写，一字不改跨镜复用）**

> 来源：Prompting Systems 时序一致性体系 + PixVerse 7项实测验证

把光线写成"物理参数"，不写成"感觉"：

| 要素 | 说明 | 写法示例 |
|------|------|---------|
| 光源位置 | 固定方向、高度 | `左上角45度` / `正上方顶光` / `逆光·镜头正对光源` |
| 光源类型 | 色温K值+灯具类型 | `2700K钨丝灯` / `5600K日光` / `冷色6500K荧光灯` / `正午5600K太阳光` |
| 光质 | 硬/柔/点光源 | `硬光·清晰投影` / `柔光·大柔光罩扩散` / `点光源·窄光束` |
| 锁死声明 | 跨镜不变声明 | `光线方向和色温在整段中保持不变` / `无动态光影变化` |

**光线锁死模板（直接套用）**：
```
场景由[位置]单一[色温K+类型]照亮，[光质描述]。光线方向固定，色温恒定。背景[明暗关系]。无额外光源变化。
```

**跨镜复用规则**：同一场景所有镜头的光线描述块一字不动复制。只在场景切换时重写光线块。

### 维度3：物品一致
关键道具的位置和状态在相邻镜头间延续。
- 上一镜掉落的武器→下一镜仍在地上
- 上一镜打开的门→下一镜仍开着（除非有人关）
- 每个道具最多写一次位置，后续镜头不改位置除非剧情有意为之

### 维度4：动作连贯
新镜头起始动作必须承接上一镜结束状态。
- 上一镜挥剑砍下→下一镜从砍中或收剑开始，不能从拔剑重新来
- 人物姿势/朝向/站位承接上镜末帧

### 维度5：台词合规
仅当前说话角色有张嘴动作。非说话角色嘴唇不动。

### 维度6：敏感过滤
全篇扫码违禁词（参考 `违禁词审查技能/SKILL.md`）。

### 维度7：持物手一致
同一人物在不同镜头中持剑/持物的手必须一致。切换手必须写清换手动作。

### 维度8：剧本上下文一致
画面情绪/人物关系/立场/角色定位不得脱离剧本上下文。
- 每镜写完后回读剧本原文，确认情绪、关系、状态三者全部对齐
- 剧本写「重伤虚弱」→不能健步如飞
- 剧本写「旧识重逢」→不能写成陌生人初见

## 四、一致性四大杠杆（Forasoft 2026 生产框架）

> 来源：Forasoft 2026 AI Video Engineering 体系

生成模型没有记忆——每次从零开始。所有一致性技术本质上都是在给模型喂比文字更强的信号，把"一个女人在红大衣里"窄化为**这个女人，在这个地方，用这个镜头运动**。

| 杠杆 | 原理 | 做法 | 光效/场景适用度 |
|------|------|------|---------------|
| **参考图锚定** | 给模型看场景/角色静帧 → 提取视觉指纹注入生成 | 上传场景锚点图，@引用 | ⭐⭐⭐ 最强·第一选择 |
| **LoRA微调** | 训练模型"认识"你的场景/角色 | 5-10张场景图训练→可复用文件 | ⭐⭐ 适合百条以上同场景 |
| **注意力共享** | 同批次帧之间互相参考，协商共享身份 | StoryDiffusion Consistent Self-Attention | ⭐⭐ 适合故事板批处理 |
| **显式参数控制** | prompt中锁定物理参数替代模糊描述 | 光线锁死四要素模板（维度2） | ⭐⭐⭐ 每次必用 |

**选择规则**：
- 偶尔用 → 参考图锚定 + Prompt锁死
- 反复用（>50条）→ LoRA + Prompt锁死
- 同批次多角度 → 注意力共享 + 参考图
- Prompt锁死是底层兜底，永远不缺

### 各模型光效/场景一致性能力对比

| 模型 | 光效保持方式 | 场景一致性评级 |
|------|------------|-------------|
| **Kling 3.0** | 多镜头Director模式·统一光线推理 | ⭐⭐⭐⭐⭐ 最强 |
| **Seedance 2.0** | @参考场景图 + Face Lock + prompt光线锁死 | ⭐⭐⭐⭐ |
| **Runway Gen-4** | 场景一致性功能·参考条件化位置 | ⭐⭐⭐⭐ |
| **Veo 3** | 身份保持prompt·长镜头光线稳定 | ⭐⭐⭐⭐ |
| **WAN 2.2** | 13种光影prompt词·图片条件化 | ⭐⭐⭐ |

## 四B、Seedance 2.0 模型级锁定技术

> 来源：Scenario官方指南 + InVideo指南 + Vmake Anchor & Master + educba + Reddit 实战

Seedance 2.0 是「条件引擎」而非纯文本模型——参考输入锚定比文字描述更可靠。

### 4.1 Reference-First 架构原理

Seedance 2.0 使用双分支扩散变换器（DiT），身份/场景数据与运动数据分离处理。参考图被当作「持久化数据对象」而非临时像素——用 @ 标签引用就是在创建全局指针。

**核心信条**：文字定义世界，图片锁定身份，视频引导运动，音频塑造节奏。

### 4.2 多模态参考槽位分配策略

使用 Seedance 2.0 的多参考能力（最多9张图 + 3段视频 + 3段音频），按功能分配：

| 槽位 | 功能 | 内容 | 复用策略 |
|------|------|------|---------|
| 图1-3 | 身份锚定 | 角色多角度参考（正面/侧面/3/4） | 所有包含该角色的片段都带这些图 |
| 图4-5 | 场景锚定 | 场景环境静帧（全景+关键锚点特写） | 所有同一场景的片段都带这些图 |
| 图6-7 | 道具/服装 | 关键道具或特殊服装参考 | 该道具/服装出现的片段才带 |
| 图8-9 | 风格/色调 | 色彩脚本或风格参考图 | 同一视觉风格的片段复用 |
| 视频1 | 运镜参考 | 想要的相机运动风格 | 同风格片段复用 |
| 视频2 | 动作参考 | 角色移动/打斗风格 | 同类型动作片段复用 |
| 音频1 | 对白/旁白 | 需要 Lip Sync 的语音 | 该段台词专用 |
| 音频2 | 环境音 | 统一场景氛围音 | 同场景复用 |

### 4.3 参考图卫生三法则

> 来源：prompt-optimizer 30.69.11

1. **角度匹配**：参考图的拍摄角度应接近目标镜头的角度（正脸参考→正脸镜头，侧脸参考→侧脸镜头）
2. **光照匹配**：参考图的光照条件应接近目标场景（白天参考→白天场景）
3. **服装匹配**：同一角色在参考图中保持同一套服装

### 4.4 Seed Locking（种子锁定）

从最满意的一次生成中记录seed值。后续同场景、同角色的生成复用该seed（或相近值），模型会从同一个「潜空间起点」出发，大幅降低场景漂移概率。

**操作**：
1. 生成第一段 → 满意 → 记录seed
2. 后续同场景片段 → 使用相同或相近seed
3. 切换场景时 → 新建场景锚点图 → 新seed
4. 记录在 Shot Ledger（逐镜记录表）中

### 4.5 Keyframe Injection（关键帧注入）

> 来源：Vmake Anchor & Master 工作流

在每段新提示词中，手动将场景锚点图作为「首帧」或「参考图」重新上传。这会在扩散过程的最初几步「提醒」模型场景空间结构。

**触发条件**：
- 第4个镜头之后（Seedance 多镜生成有注意力衰减）
- 切换景别幅度大时（大全景→大特写）
- 连续生成超过3段之后

### 4.6 Reset in Chains（链条重置）

> 来源：prompt-optimizer 30.47

身份漂移是逐段累积的：第1段100%匹配，第2段有细微偏差，第4段已明显不同。

**方法**：每生成4-5段，回到原始场景参考图重新生成一段作为「校准原点」，打断漂移累积链。

### 4.7 环境描述前20-30词权重规则

> 来源：InVideo指南

Seedance 2.0 的前20-30个词权重最高，用于锁定主题和核心动作。环境描述必须尽早出现。

**写法**：
- ✅ `户外石质演武台，正午日光。角色A从画左冲入…`
- ❌ `角色A从画左冲入，他和角色B对峙，这是一场激烈的打斗，场景设在户外石质演武台…`

## 五、多镜头一致性工作流

> 来源：prompt-optimizer 30.69.13-14（Crepal Dora 42条序列验证）

### Step 1：固定场景块（Fixed Scene Block）

创建一个独立文本块，包含场景的所有不可变锚点：

```
【场景锚点块 - 演武台】
- 空间：圆形石质演武台，直径约15米，台面微磨损
- 锚点：北→主殿红柱金瓦，东→歪脖子树挂铜铃，西→石阶通观战台
- 光线：正午日光从画右上（东南方向）斜射，地面泛暖白反光
- 色调：暖金色调，高对比度，硬阴影
```

跨镜复用，只改一次全段生效。

### Step 2：按光线条件分批生成（批次策略）

> 来源：AIMagicX 2026 长视频一致性生产体系

**不按剧本顺序生成，按光线条件分批。** 同批用同一套参考图 + 同一段光线描述。

| 批次 | 光线条件 | 包含镜头 | 共享资源 |
|------|---------|---------|---------|
| Batch 1 | 暖光室内 | 所有"2700K钨丝灯室内"的镜头 | 同场景参考图 + 同seed + 同光线块 |
| Batch 2 | 冷光室外 | 所有"5600K日光户外"的镜头 | 同上 |
| Batch 3 | 夜景 | 所有"月光+火光混合"的镜头 | 同上 |
| ... | ... | ... | ... |

**为什么分批有效**：同光线条件下，模型的潜空间采样分布更集中，跨镜变异幅度更小。"近景·暖光"→"全景·暖光"的漂移远小于"近景·暖光"→"全景·冷光"。

**分批后仍用Shot Ledger逐镜记录**（见下方Step 3）。

### Step 3：逐镜记录表（Shot Ledger）

| 镜号 | 场景块 | 参考图 | seed | 批次 | 生成状态 | 备注 |
|------|--------|--------|------|------|---------|------|
| Shot 1 | 演武台 | 图4(全景) | 12345 | Batch2·日光 | ✅ | 锚点全部可见 |
| Shot 2 | 演武台 | 图4(全景) | 12345 | Batch2·日光 | ✅ |  |
| Shot 3 | 演武台 | 图4(全景)+图5(特写) | 12345 | Batch2·日光 | ⚠️ | 注意光方向 |
| ... |

### Step 4：一致性复查（Consistency Pass）

全部分镜写完后，逐镜对比场景锚点块：

- [ ] 场景块中的锚点元素在分镜内容中是否出现？
- [ ] seed值是否一致？
- [ ] 参考图是否复用同一组场景图？
- [ ] 四镜循环播放检查光影是否跳变？

### Step 5：漂移修复决策阶梯

```
发现场景漂移 → 判断严重程度：

轻度（单镜，周围正常）→ 该镜补写锚点元素，微调内容重生成
  
中度（局部元素丢失，如某家具消失）→ 在该镜提示词中显式写入丢失元素
  
重度（场景整体漂移，光影/结构/色调全变）→ 重新上传场景参考图 + 锁定seed + 重写场景锚点块
```

**原则**：修复成本 < 从头重生成 → 修复。否则 → 严格锚点重生成。

## 六、场景类型特定一致性策略

### 户外场景
- **防漂移要点**：天空色调、云层密度、远景地标、地面材质
- **提示词写法**：在【场景】段锁定「户外[地点]，[时间]，[天气]，[光线方向]，远景可见[地标]」
- **每镜必带**：地面材质和光线方向

### 室内场景
- **防漂移要点**：墙壁颜色、家具位置、门窗方位、天花板高度
- **提示词写法**：在【场景】段锁定四向特征
- **每镜必带**：至少一个墙面锚点元素在背景中出现

### 夜景/暗光场景
- **防漂移要点**：光源数量、光源类型（月光/灯光/火光）、色温
- **提示词写法**：锁定「主光源[类型]，[颜色]，[方向]，辅光[类型]」
- **每镜必带**：光源可见（或光影效果可见）

### 打斗/动作场景
- **防漂移要点**：地面破坏状态延续、扬尘/烟雾持续、血迹/痕迹不消失
- **提示词写法**：在内容中每镜锚定「地面[状态]，[破坏痕迹]，[环境元素运动]」
- **每镜必带**：至少一处环境元素在动（扬尘/碎石/落叶/火星）

## 七、出稿前场景一致性检查清单

> 整合自四面墙/八维/多镜头工作流/SelfCheck 全部去重后

### 空间逻辑（四面墙）
- [ ] 光源方向每镜一致
- [ ] 场景锚点元素每镜至少出现1个
- [ ] 人物移动路径连贯
- [ ] 远景/近景空间关系匹配
- [ ] 物体遮挡关系合理
- [ ] 🔴 每镜人物面朝方向与相机方向不重合（人物朝向+机位方位已绑定写）

### 跨镜连续性（八维）
- [ ] 人物/服装/道具一致（参考图未变）
- [ ] 时间/地点/光线无缝衔接
- [ ] 关键道具位置延续
- [ ] 动作起止承接上镜
- [ ] 持物手一致
- [ ] 情绪/关系/状态对齐剧本

### 模型锁定（Seedance）
- [ ] 场景参考图复用（同场景同组图）
- [ ] seed值锁定（同场景同seed）
- [ ] 前20-30词含环境描述
- [ ] 超过第4镜后注入了场景锚点
- [ ] 超过4段后执行了Reset in Chains

### 违禁/质量
- [ ] 全文扫码违禁词
- [ ] avoid行含防漂移约束 `避免场景漂移，避免光影跳变，避免物体形变消失`

## 八、场景漂移修复速查

| 症状 | 最可能原因 | 最快修复 |
|------|-----------|---------|
| 背景变了 | 场景参考图未复用 | 重新上传同一张场景静帧作为参考 |
| 光影跳了 | 光源描述不一致 | 锁定光源方向+色温到场景锚点块 |
| 物体消失 | 物体无参考图且文本描述弱 | 加道具参考图或强化物体描述 |
| 色调变了 | 风格参考图缺失或seed变了 | 锁定风格参考图+seed |
| 空间结构错乱 | 四面墙规则被打破 | 回读场景锚点块+修正矛盾镜头 |
| 第4镜后开始漂移 | 注意力衰减 | 第4镜注入场景锚点Keyframe |
