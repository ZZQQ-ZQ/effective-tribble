# 重叠判定与整合规则

## 一、扫描触发条件

以下任一表达触发技能门禁审查：

| 触发词 | 示例 |
|--------|------|
| "创建技能" / "新建技能" | "我创建一个新技能" |
| "写个技能" / "做个技能" | "帮我做个XXX的技能" |
| "技能化" / "封装成技能" | "把这个规则技能化" |
| "加到技能里" | "把这个加到技能里" |

## 二、扫描流程

### Step 1：收集新技能信息

在开始扫描前，确认：
- 技能名称/方向
- 核心功能（一句话）
- 触发场景（用户在什么情况下会用到）

### Step 2：三层扫描

按优先级遍历三层技能目录：

```
层1: ~/.openclaw/workspace/skills/*/SKILL.md          （工作区，主维护区）
层2: ~/.openclaw/skills/*/SKILL.md                    （用户全局）
层3: /usr/lib/node_modules/openclaw/skills/*/SKILL.md （系统内置，只读）
```

### Step 3：匹配算法

对每个已有技能，按以下维度逐一比对新技能：

| 维度 | 权重 | 判定方法 |
|------|------|---------|
| 功能关键词 | 🔴高 | 提取 name + description 中的核心名词/动词，与新技能交叉比对 |
| 触发场景 | 🔴高 | 用户会在什么场景下说触发词？场景重叠=大概率重复 |
| 产出物 | 🟡中 | 两个技能输出相同类型的产物（如都是提示词、都是报告） |
| 受众/用途 | 🟢低 | 虽然功能相似但面向不同角色或平台 |

### Step 4：判定等级

| 等级 | 条件 | 操作 |
|------|------|------|
| 🔴 高度重叠 | 功能关键词+触发场景同时匹配 | 拦截创建，整合进已有技能 |
| 🟡 部分重叠 | 功能关键词匹配但触发场景不同 | 提示用户选择：扩建旧技能 or 独立创建 |
| 🟢 互补关系 | 不同但可协作 | 建议新建，在 SKILL.md 中注明与相关技能的协作关系 |
| ⚪ 无关联 | 完全独立 | 放行，正常创建 |

## 三、整合操作

### 确定整合目标技能后：

1. **读旧技能 SKILL.md** → 理解现有结构和内容
2. **判断新知识的归属位置**：
   - 核心流程 → 写入 SKILL.md 正文
   - 参考细节 → 新建/追加 references/ 文件
   - 脚本工具 → 追加到 scripts/ 目录
3. **更新旧技能**：
   - 追加新章节或扩展现有章节
   - 更新 frontmatter description 包含新触发场景
   - 不破坏原有结构
4. **更新索引**：
   - 更新 `references/skill-map.md` 中的技能描述
   - 更新 MEMORY.md 中的技能索引条目
5. **报告整合结果**：告知用户已整合到哪个技能、追加了什么内容

## 四、特例

以下情况即使功能重叠也允许独立创建：

- 新技能面向**完全不同的平台/工具**（如 Midjourney vs Seedance）
- 旧技能已标记 DEPRECATED
- 旧技能为系统内置只读，且不适合上层覆盖
- 用户明确要求独立创建（需确认两次）
