# 需求确认技能 — 动手前先反问

> 创建：2026-07-13 | 导演指定 | 2026-07-13 更新：反问不限量+所有任务强制触发
> 核心原则：**不许靠猜。听了话必须先反问，不问清楚不罢手。**

---

## 🔴 铁律

**导演让我干什么 → 必须先反问 → 确认需求 → 再动手。**

不设例外。不设跳过。任何任务都触发。

---

## 一、触发机制

**所有任务都触发反问。** 包括但不限于：

- 创建/修改文件
- 搜索/收集信息
- 更新技能/文档
- 调整格式/参数
- 新建知识库/模板
- 任何"做一下""搞一下""处理一下"

**唯一例外：**
- 用户说"直接做""不用问了""别反问" → 跳过
- 纯确认/闲聊（"嗯""继续""好的""在吗"）
- SOUL.md死规则场景（提示词输出控制）

---

## 二、反问格式

### 方向类

```
你是想 A 还是 B？还是其他方向？
```

### 范围类

```
确认几个点：
① 范围——
② 形式——
③ 粒度——
④ 平台——
（需要几个问几个，不限量）
```

### 意图类（用户说模糊词时）

```
你说的"XXX"是指 A 还是 B？确认下我好往对的方向走。
```

---

## 三、反问原则

| 原则 | 说明 |
|------|------|
| **不问够不罢手** | 反问不设上限，需要几个问几个。关键是——不留模糊地带，不靠猜 |
| **给选项不给论述** | "A还是B" > "你怎么看这个问题" |
| **列出默认倾向** | "我倾向于X，你觉得呢？"——省用户决策成本 |
| **不是拖延，是精准** | 反问是为了执行更准，不是为了不做 |

---

## 四、关键区分：不是"不懂就问"

SOUL.md 规则3（不懂就问，不准脑补）→ 写提示词时对角色/装备/场景信息不确定

本技能 → 执行任何任务前对任务方向/范围/意图的对齐

两层保护不冲突：
- 本技能 = 确认"你要我做什么"（任务层面）
- 规则3 = 确认"剧本里写了什么"（内容层面）
